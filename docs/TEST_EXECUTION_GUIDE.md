## 测试执行说明

### 一、测试依据

以项目根目录下 `docs/test_plan.md` 文档为核心测试依据，覆盖文档中定义的所有功能模块、交互规则、性能指标及兼容性要求。  
测试环境：Ubuntu 20.04/22.04 服务器（Nginx + FastAPI + PostgreSQL）、企业微信沙箱环境。

### 二、测试范围（全站）

1. **核心功能模块**
   - **设备使用/维护登记模块**：扫码登记、手动输入设备编号登记、记录提交/撤销；
   - **设备管理模块**：新增/编辑/停用/删除设备、二维码查看、状态管理；
   - **登记记录查询模块**：多条件筛选、数据导出、列排序/空值展示优化；
   - **我的登记记录模块**：列表加载、筛选、撤销操作；
   - **审计日志模块**：筛选、搜索、导出、日志展示；
   - **权限管理模块**：医工部/技师/护士角色权限校验。

2. **访问入口**
   - **PC 端**：Chrome / Firefox / Edge / IE 兼容测试；
   - **企业微信端**：移动端适配、扫码功能、操作流程。

3. **非功能测试**
   - **性能**：页面加载时长（≤ 3s）、接口响应时长（≤ 1s）、大数据量（1000+ 条日志/记录）加载稳定性；
   - **兼容性**：Ubuntu 20.04/22.04 服务器部署后 Nginx 反向代理兼容性、不同分辨率 PC 端适配；
   - **安全性**：接口鉴权、敏感操作（删除/停用）二次确认、日志审计完整性；
   - **易用性**：UI 交互优化点验证（滚动条固定、标签样式、提示文字、按钮反馈）。

### 三、测试执行要求

1. 严格按照 `docs/test_plan.md` 中定义的测试用例模板、优先级、预期结果执行。
2. 覆盖正向/反向用例：
   - **正向**：合法设备编号登记、权限内操作（如护士提交使用记录）；
   - **反向**：非法设备编号登记、权限越界操作（如护士尝试删除设备）、重复提交记录。
3. 验证所有 UI/交互优化点落地效果（如表格奇偶行背景、空值提示、滚动条固定）。
4. 记录测试过程中与 `docs/test_plan.md` 不符的功能点、UI 问题、性能瓶颈。
5. 输出测试报告，包含：
   - 用例执行统计表格；
   - 问题清单（含截图链接）、优化点验证结果表；
   - 明确问题分级（致命 / 严重 / 一般 / 轻微）、复现步骤及截图。
